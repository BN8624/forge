# Forge 구축 체크리스트

- [x] 독립 프로젝트 경계와 5권 목표를 문서화한다.
- [x] Atelier에서 공통 실행 인프라와 최소 참고자료만 복사한다.
- [x] 시리즈, 권, 사건, 장면, 상태 원장 스키마를 정의한다.
- [x] 소유권과 계층 참조를 검사하는 구조 검증기를 구현한다.
- [x] 정상 및 실패 픽스처로 검증기를 테스트한다.
- [x] 프로젝트를 `C:\Users\USER\forge` 독립 경로로 분리한다.
- [x] 독립 Git 저장소와 GitHub 공개 저장소를 연결한다.
- [x] JSON Schema를 구조 검증기에 연결하고 계약 위반 테스트를 추가한다.
- [ ] 구조 문서 후보 생성기를 구현한다.
- [x] 독립 검증과 정본 승격의 원자적 흐름을 구현한다.
  - [x] 후보 `story`를 현재 정본과 분리된 임시 스냅샷에서 검증한다.
  - [x] 검증 실패 또는 교체 오류 시 기존 `story`가 보존되는지 테스트한다.
  - [x] 중단된 교체를 다음 실행에서 복구한다.
  - [x] 검증을 통과한 후보만 `story` 정본으로 승격한다.
- [x] 상태 원장 갱신과 재실행 검증을 구현한다.
  - [x] 검증된 구조에서 최종 장면, 상태, 적용 요소를 결정적으로 파생한다.
  - [x] 원장을 원자적으로 저장하고 실패 시 기존 파일을 보존한다.
  - [x] 같은 구조로 재실행한 원장 파일이 바이트 단위로 같은지 검사한다.
  - [x] 구조 검증기가 적용 요소 목록의 불일치도 거부한다.
- [ ] 5권 신규 초안 생성을 시작한다.
